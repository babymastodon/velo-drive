# VeloDrive (legacy) ‚Äî User Goals / End-to-End Journeys

Scour of the legacy vanilla-JS cycling-workout PWA at `docs/`. This catalogs every distinct thing a user sets out to accomplish, the ordered steps, the key branches/variants, and the legacy code that implements each journey.

**41 journeys** across 6 areas: Onboarding/Config (8), Hardware (3), Ride Execution (11), Import (4), Build/Edit (6), Library & Planner (9).

The single most important journeys are: **J17 Ride a structured workout (ERG)**, **J18 Auto-start by pedaling**, **J19/J20 Auto-pause/auto-resume**, **J24 Finish & export FIT**, **J27 Crash recovery (resumes paused)**, **J10 Import via extension scrape + J11 scrape-on-focus**, and **J37 Plan a training week**. The app revolves around a single `CanonicalWorkout` format (segments + textEvents) shared by every source/sink, persisted to a local File System Access folder (`workouts/`, `history/`, `trash/`, `schedule.json`) plus IndexedDB for settings/state.

Architecture spine: `workout.js` is the page shell (boot, keyboard, cross-cutting auto-behaviors); `workout-engine.js` is the 1 Hz ride state machine; `ble-manager.js` is BLE/FTMS; `workout-picker.js` hosts both the library and the in-modal builder; `workout-planner.js` is the calendar; `storage.js` is persistence.
