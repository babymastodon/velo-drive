# Adversarial Review — VeloDrive rewrite (`web/src`)

Scope: read-only audit of `web/src` (~18k LOC, 40 files). The stated goal is
"lightweight, simple, straightforward, well-encapsulated" code that replaces a
spaghetti legacy. This review attacks whether that goal was met. Every claim
cites `file:line` evidence. Severity is **as the reviewer sees it**, deliberately
harsh; filter accordingly.

The one-line verdict: **the rewrite is cleaner-looking than the legacy but it is
NOT lightweight or simple — it is a line-for-line transliteration of the legacy
that preserves the legacy's worst structural sins (a god-object builder model, a
positional tuple data shape touched in 10 files, dual theme machinery, a tick-
counted clock) and adds new web-specific foot-guns (Svelte-proxy de-proxying,
two theme version counters, a non-reactive overlay key-handler registry). The
abstractions are mostly honest, but several are illusory.** The pervasive
"mirrors docs/…:NNNN" comments are an admission: this is a port, and the
spaghetti was ported with it.

## If I could only fix 5 things

Ranked by real damage to maintainability/correctness, not nitpicks.

1. **Kill the `JSON.parse(JSON.stringify())` de-proxy (finding #5, `WebFileStore.ts:213`).**
   This is the highest-risk single line: it silently corrupts/strips any
   non-JSON field on every settings + active-ride write, and the test suite is
   *by its own admission* blind to it. Replace with `structuredClone($state.snapshot(value))`
   and add a real-IndexedDB integration test. One line, but it guards every
   persisted byte.

2. **Encapsulate the `RawSegment` tuple and delete the 6 copies of `getRawCadence`
   (finding #6).** Create `core/segment.ts` with `segmentMinutes/StartPct/EndPct/
   cadence/isFreeRide` accessors and route all 10 files through it. This is the
   biggest "well-encapsulated" win available and removes the most duplicated,
   drift-prone code in the app (cadence parsing × 6, NP loop × 2 per finding #12).

3. **Decompose `builder-backend.ts` and fix the `rampUp`/`rampDown` type lie
   (findings #1, #2).** At minimum split selection-state (the 5 coupled fields)
   into its own documented sub-module with a stated invariant, and make
   `BlockKind` actually include the kinds the code stores so the `as BlockKind` /
   `as string` casts disappear. This is the file a new engineer will curse first.

4. **Collapse the dual theme machinery and the two version counters (finding #4).**
   One theme module; one redraw signal. The `version` vs `autoVersion` split
   exists solely to preserve legacy screenshot baselines — fix the baselines, not
   the runtime. Remove the "read a getter to install a global observer" pattern in
   favor of an explicit `initTheme()` at boot.

5. **Unify Escape/keymap routing and the overlay handler registry (findings #7, #16).**
   Make `overlayKeyHandlers` reactive (or replace the registry with a single
   keymap table + an `activeOverlay` switch), and have ONE owner of the Escape
   disposition instead of the current two (`App.onKeydown` + `UiStore.handleEscape`,
   with its self-described possibly-dead "defensive guard" branches).

### Honorable-mention cleanups (cheap, high signal)
- Fix the lying `DEFERRED` comment at `PickerView.svelte:8-9` (the builder ships).
- Either honor the `FileStore` port or delete it and type the UI against
  `WebFileStore` honestly (finding #9) — the half-port is worst of both worlds.
- `STATS_CACHE_VERSION = 30` (finding #10): derive the cache key from a hash of
  the preview shape, or at least add a CI assert that the version bumps when the
  `StatsCacheEntry` interface changes.
