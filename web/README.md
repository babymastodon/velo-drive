# VeloDrive — Web (TypeScript + Vite + Svelte 5 rewrite)

A modern, type-safe rewrite of the VeloDrive vanilla-JS app (`../docs`), migrated
view-by-view against a **shimmed copy of the legacy code** with every view gated by
real functional **and** visual screenshot tests.

## How it's built

- **`src/core/`** — framework-agnostic TS: the canonical workout `model`, `zwo`/`fit`
  codecs, `metrics`, the ride `engine`, `beeper`, `chart` renderers, `builder-backend`.
  Validated by **differential tests** that assert byte/value-equality with the legacy
  `docs/*.js` over the 41 bundled workouts (the legacy code is the oracle).
- **`src/ports/`** — two platform seams: `TrainerTransport` (Web Bluetooth FTMS) and
  `FileStore` (File System Access + IndexedDB), wrapping the existing behavior behind
  typed interfaces. These are the seams an eventual Tauri port swaps.
- **`src/state/`** — Svelte 5 signals (engine VM, overlay/ui host, dialog).
- **`src/ui/`** — Svelte components reproducing the legacy DOM **classes + `data-testid`s**
  so the legacy CSS — re-hosted **verbatim and global** in `src/styles/` — applies unchanged.

## The test harness (`harness/`, `tests/`)

Hermetic and deterministic — no real hardware, filesystem, clock, or network:

- **Shims:** `scripts/apply-shims.mjs` injects one `velo-shim.js` into a copy of the legacy
  app (`legacy-shimmed/`, regenerated by `scripts/setup-legacy.sh`) that swaps platform
  globals from `window.__VELO_TEST_ENV__` — no module edits, a no-op without injection.
- **Fakes:** a **FTMS trainer simulator** (validated against the legacy byte parser), an
  in-memory filesystem, a virtual-time clock (steps a ride in ms), an audio recorder.
- **Real visual gate** (`tests/visual/compare.ts`): `compareImages()` pixel-diffs the new
  render against the legacy baseline and **fails on divergence** (it caught — and forced a
  fix of — a 4.85% HUD layout regression during the migration). A gate self-test proves the
  comparator discriminates (identical → 0, vs a solid image → 0.99). **Not** capture-only.
- The **same harness drives both** the shimmed legacy app and the new Svelte app, so every
  `*.new.spec.ts` diffs against a `*.legacy.spec.ts` baseline captured in the identical state.

## Commands

```sh
npm install
bash scripts/setup-legacy.sh          # regenerate legacy/ + legacy-shimmed/ from ../docs
npm run dev                            # run the new app
npm run typecheck                      # tsc --noEmit (strict)
npm run test                           # vitest: unit / parity / differential
npx playwright install chromium        # once
npx playwright test                    # e2e: legacy baselines + new-app behavior + visual diffs
```

## Documented simplifications (sub-features, not core flows)

These legacy sub-features render with full DOM/visual parity but are intentionally not
wired in this migration pass (tracked for follow-up):

- **Planner:** drag-to-reschedule, the picker schedule-mode handoff (replaced by a Dialog),
  and deep-scroll calendar row-recycling (initial 16-week window renders identically).
- **Builder:** block-reorder-by-drag and text-event drag (click-select instead); clipboard
  copy/paste and TrainerDay-URL / file-upload import buttons are rendered but inert.
- **Known residual:** the planner shows a ~1px uniform vertical text offset vs legacy
  (under the visual gate; flagged for polish).

See `../docs-analysis/05-design-and-migration-plan.md` for the full design + plan.
